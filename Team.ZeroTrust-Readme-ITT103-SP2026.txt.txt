Papers of the Program

This paper presents a console-based Point of Sale (POS) system designed for "Team Zero Trust's Best Buy Retail Store." The system manages an inventory of 11 products, allowing users to add or remove items from a shopping cart with real-time stock validation. It implements automated business logic including a 5% discount on subtotals exceeding $5,000 and a 10% tax calculation. The checkout module processes cash payments, verifies sufficient funds, and calculates change. The system operates through a menu-driven loop, supports multiple transactions per session, and includes error handling for invalid inputs, out-of-stock items, and empty carts.


how to run a program
The program displays a menu with 6 options. The user selects an option by entering a number (1-6). They can view products, add items to cart (which checks stock and updates inventory), remove items from cart, view cart with subtotal, checkout (which applies discount over $5000 and 10% tax, then processes payment), or cancel the transaction. After checkout or cancel, the program asks if the user wants to start a new transaction. If yes, the cart resets; if no, the program ends with a goodbye message.


Here's a short summary of the assumptions and limitations of the program:

Assumptions:
User will enter numeric values for quantity and payment

User knows product names exactly (case-insensitive but spelled correctly)

All transactions use cash payment only

Discount applies automatically when subtotal exceeds $5,000

Tax rate is fixed at 10%

Limitations:
No data persistence – Inventory resets when program restarts

No receipt printing – Transaction details shown only on screen

Cash only – No credit card, debit, or digital payments

Single user only – No multi-user or employee login system

No transaction history – Past sales are not logged or tracked

Basic error handling – No try-catch for non-integer inputs in some areas

No returns/refunds – Once checked out, cannot reverse transaction
